# Changelog

## [Unreleased]
### 新增
- 新增输入框批量处理流程  
  自动分条  
  歧义分条确认  
  手动分条编辑
- 新增批量进度事件 `BatchProgressEvent`  
  统一包含 `index` `total` `stage` `message`
- 新增批量取消机制  
  可中断正在运行任务  
  未完成条目标记 `cancelled`
- 新增批量结果分区展示  
  成功  
  待确认  
  失败
- 新增分条结果结构 `reason_code`

### 优化
- 检索和候选确认走后台线程  
  运行中可继续浏览已完成结果
- 检索超时控制增强  
  默认 `search_timeout_sec=6.0`
- Scholar 入口去重  
  待确认时仅显示“在 Scholar 打开当前候选”
- 结果区块顺序调整为  
  成功在前  
  待确认在中  
  失败在后
- DOI 路径增加多层容错和回退检索
- 失败条目显示增强  
  回填标题 作者 年份 DOI 和失败说明

### 修复
- 修复批量中大量条目因 DOI 请求异常直接失败的问题
- 修复失败条目只显示编号不显示内容的问题
- 修复切换 BibTeX key 规则后内容不更新的问题  
  现在优先从 BibTeX 解析标题 作者 年份 DOI，再回退 selected
- 修复中文 `J/OL` 风格参考文献解析不稳定的问题

### 测试
- 新增批量分条测试
- 新增批量取消与顺序回归测试
- 新增 BibTeX key 切换回归测试
- 本地通过 `pytest` 全量回归

## [0.1.2] - 2026-04-20
### 修复
- 修复 Vancouver 风格参考文献识别，避免将作者段误判为标题输入
- 修复年份在末尾时的解析逻辑，正确提取作者 标题 年份
- 修复部分参考文献在检索不稳定时直接失败的问题  
  现在会提供可确认的无 DOI 兜底候选
- 修复 BibTeX 输出偶发单行问题  
  规范化 `month=june` 等字段后统一输出多行
- 修复 Crossref 请求异常导致的检索中断  
  请求失败时自动回退为空结果并继续流程
- 修复候选自动通过规则过严的问题  
  对明显领先的 DOI 候选支持自动成功

### 测试
- 增加并通过回归测试  
  覆盖参考文献识别、解析、匹配和 BibTeX 格式化

## [0.1.1] - 2026-04-12
### 修复
- 修复 BibTeX key 规则 `作者姓+年份+标题首词` 中标题首词首字母为小写的问题  
  输出统一为 `Zhou2007Functional`

## [0.1.0] - 2026-04-11
### 新增
- 单条输入 MVP
- 支持 DOI、标题、完整参考文献字符串输入
- 支持 Crossref 和 OpenAlex 双源检索
- 支持 BibTeX 生成、复制和 key 规则切换
